# 商品上架与SKU管理 (PDD专精)

## 📌 核心策略
拼多多的上架不是简单的填信息，而是一套**价格+SKU+主图**的组合策略。

## 🔑 PDD标题规则（必记）
- 上限30个汉字
- 权重顺序：核心词 > 属性词 > 场景词 > 卖点词
- 不可堆砌重复词，否则降权
- 标题格式：**核心词 + 属性词 + 卖点词 + 场景词**
- 示例：`纯棉短袖T恤女夏季透气宽松打底衫学生百搭`

## 🏷 标题公式
```markdown
公式1（精准型）：核心词 + 类目词 + 核心属性 + 场景
公式2（引流型）：热搜词 + 核心词 + 价格锚点 + 卖点
公式3（长尾型）：长尾搜索词 + 核心词 + 人群词 + 风格词
```

## 📐 SKU策略（拼多多最重要的一环）
### 黄金SKU结构
```
引流SKU → 最低价，不赚钱但要能跑量
├── 规格少，价格极低（比如一件装/基础款）
├── 用于拉搜索权重和活动门槛
└── 注意：不能亏太多，控制在成本±5%

主推SKU → 利润款，大多数人会买
├── 规格适中，价格适中
├── 承担主要利润来源
└── 占总销量的60%以上

高价SKU → 锚点款，衬托主推款划算
├── 规格多，价格高
├── 作用：让主推SKU看起来性价比高
└── 可以不怎么卖，但不能没有
```

### SKU命名技巧
- 不要用"基础款/升级款/尊享款"这种烂大街
- 用具体规格命名：`1袋体验装 / 3袋实惠装 / 5袋超值装`
- 引流SKU名字要突出"试用/体验"感
- 主推SKU名字要突出"最划算"信息

### 拼多多SKU禁忌
- ❌ 不同SKU价格差距过大（超过5倍容易被判定异常）
- ❌ SKU图片与实物不符（罚款重灾区）
- ❌ 虚假低价引流（主图价格与最低价不符 → 限流）
- ✅ 最低价SKU库存要够（卖完没及时补→权重掉）

## 🖼 主图策略
```
首图（决定点击率）
├── 核心信息：产品主体 + 价格/优惠 + 卖点
├── 风格：简洁/突出/有冲击力
├── 色彩：高饱和度，与竞品差异明显
└── 避雷：不要满屏文字

附图2-5
├── 图2：卖点图（3-4个核心卖点配图标）
├── 图3：细节/材质图
├── 图4：场景/效果图
└── 图5：尺码/规格/对比图
```

## 🎯 定价策略
### 拼多多价格锚定法
```
市场均价        → 参考同行TOP100的价格中位数
引流价          → 市场均价的60-70%（亏本或微亏）
主推价          → 市场均价的80-90%（利润主要来源）
高价锚点        → 市场均价的120-150%（衬托主推款）
```

### 价格敏感度判断
- **9.9以下**：冲动消费，重视觉轻描述
- **9.9-29.9**：性价比驱动，重评价和销量
- **29.9-99.9**：理性消费，重详情和品牌
- **100以上**：品质驱动，重信任感

## 📋 输出规范
每次上架输出一个完整方案，包含：
1. 3个标题版本
2. SKU结构表（引流/主推/锚点）
3. 5张主图方案
4. 价格策略建议
5. 关键词清单（10-15个热搜词）
6. 运费模板建议

## ⚠️ 上架检查清单
- [ ] 标题字数≤30
- [ ] 主图无水印/二维码
- [ ] 最低价SKU库存充足(≥50)
- [ ] SKU图片与本SKU一致
- [ ] 价格无异常波动
- [ ] 详情页无极限词（最好/第一/绝对）
- [ ] 详情页800px宽以上
- [ ] 商品属性填完整（提高搜索权重）
