# OpenBusiness 🏪 —— 中小商家经营助手

> 电商从业者 & 实体生意人的日常经营实用程序
> 核心理念：低成本、易操作、高频使用

---

## 一、产品定位

**一句话**：一个跑在微信里的「生意管家」，帮小店老板搞定每日经营琐事。

**目标用户**：
- 电商卖家（淘宝/拼多多/抖音/快手/京东）
- 实体店主（餐饮/零售/服务类）
- 微商/私域从业者

**核心差异**：
- ✅ 不装 App —— 微信就能用
- ✅ 不花大钱 —— 免费起步，用到的 API 自己可控
- ✅ 不学操作 —— 自然语言对话式交互
- ✅ 不断联 —— 离线也能看本地数据

## 二、核心功能模块（MVP → 完整版）

### 🥇 MVP（第1周可用）

```
┌─────────────────────────────────────────┐
│             生意看板                      │
│  今日营收  │  待发货  │  售后待处理      │
│  ¥ 3,280   │    12   │     3            │
│                                          │
│  ┌──────────────────────────────────┐    │
│  │  近7天销售额趋势                  │    │
│  │  ██▄██▆███▇███▅███▄███▃██      │    │
│  └──────────────────────────────────┘    │
└─────────────────────────────────────────┘
```

**模块 1：每日生意看板**
- 多平台订单汇总（淘宝/拼多多/抖音）
- 今日营收、待发货、售后单数量
- 近7天趋势简图

**模块 2：快捷记账**
- 自然语言记账：「今天进了200块的货」
- 分类：进货/人工/房租/物流/营销/其他
- 每日收支汇总、利润自动算

**模块 3：待办清单**
- 每天自动生成待办：「3个售后未处理」「2单超时未发货」
- 手动添加待办：「明天去批发市场」
- 完成后划掉

### 🔄 第2周

**模块 4：智能备忘录**
- 客户备注：「王姐要的M码粉色，到了通知」
- 供应商信息：「义乌老张，袜子批发，电话13xxx」
- 知识库：行业经验、话术、配方等

**模块 5：快速查件**
- 输入单号查物流
- 批量查询
- 异常提醒（滞留/退回）

### ⚡ 第3周

**模块 6：消息聚合**
- 各平台消息提醒聚合
- AI 自动分类：重要客户/售后/广告
- 一键回复模板

**模块 7：低库存预警**
- 设定安全库存
- 低于阈值自动提醒
- 可一键生成采购清单

### 🚀 第4周

**模块 8：商品管理**
- 简易商品库
- 批量改价/改库存
- 自动同步（可选）

**模块 9：客户运营**
- 高频客户标记
- 回头客统计
- 回访提醒

## 三、交互方式

```
┌──────────────────────────────┐
│  ⏰ 09:00  早安！今日提醒      │
│                              │
│  今日待办                    │
│  ☐ 处理售后单 ×3              │
│  ☐ 检查库存：A款低于安全库存  │
│  ☑ 已发货 2单 → 已更新物流    │
│                              │
│  昨日营收 ¥2,860              │
│  净利  +¥1,240（43%）          │
│                              │
│  ─── 说点什么？ ───           │
│  [  输入框...              ] │
│  [查看今日] [记账] [查件]    │
└──────────────────────────────┘
```

**三种使用模式**：

1. **主动推送**（早报/提醒）
   - 每日9点推送生意简报
   - 异常实时提醒（售后/超时/库存预警）
   - 好消息播报（爆单通知 🎉）

2. **对话查询**（问什么答什么）
   - 「今天赚了多少？」
   - 「A款还有多少库存？」
   - 「帮我查一下SF1234567890」

3. **快捷指令**（一键操作）
   - 「记账：进货30件 ¥600」
   - 「新增待办：明天去市场」
   - 「生成采购清单」

## 四、技术架构

### 方案 A：纯本地（0成本，推荐MVP）

```
┌────────────────────────────────────┐
│           微信 / Telegram           │  ← 交互层
├────────────────────────────────────┤
│         OpenClaw (奥创)             │  ← AI 调度层
│    ┌──────────────────────────┐    │
│    │  意图识别 + 任务编排      │    │
│    └──────────────────────────┘    │
├────────────────────────────────────┤
│          工具模块                    │
│  ┌──────┐ ┌──────┐ ┌──────────┐  │
│  │ 记账  │ │ 备忘录 │ │ 物流查询  │  │
│  ├──────┤ ├──────┤ ├──────────┤  │
│  │ 库存  │ │ 商品库 │ │ 待办事项  │  │
│  └──────┘ └──────┘ └──────────┘  │
├────────────────────────────────────┤
│          数据存储                    │
│  ┌──────────────────────────┐      │
│  │  本地 SQLite / JSON 文件   │      │  ← 不用服务器
│  └──────────────────────────┘      │
├────────────────────────────────────┤
│      外部 API（按需开）             │
│  ┌──────┐ ┌──────┐ ┌──────┐      │
│  │物流  │ │ 平台  │ │天气  │      │
│  │接口  │ │开放平 │ │       │      │
│  │      │ │台     │ │       │      │
│  └──────┘ └──────┘ └──────┘      │
└────────────────────────────────────┘
```

**成本**：¥0/月（仅消耗你的 API 额度，极低）
**依赖**：Node.js + OpenClaw + SQLite
**上线时间**：3-5天出 MVP

### 方案 B：轻量服务器版（多人/多店）

- 同上 + 一个云服务器（最低配 ¥30-50/月）
- 支持多用户、多店铺
- 支持定时任务（自动抓单、自动推送）

## 六、为什么能用 OpenClaw + Codex 做

| 组件 | 作用 |
|------|------|
| **OpenClaw** | 对话入口 + 意图理解 + 任务调度 + 定时触发 |
| **TaskFlow** | 多步骤业务流（下单→记账→更新库存→通知） |
| **Cron** | 每日早报、定时检查、预警推送 |
| **SQLite** | 本地数据持久化，无需服务器 |
| **Codex（可选）** | 写脚本、数据分析、自动修复数据问题 |

**一个完整流程的例子**：

> 用户说：「来了个退货，A款红色M码一件，¥129」
>
> 奥创自动：
> 1. ✅ 记入售后单（待处理）
> 2. ✅ 更新库存（+1件可售）
> 3. ✅ 检查该客户退款率，标记高频售后
> 4. ✅ 回复：「已记录，售后单#038，预计3天到仓检查后退款」
